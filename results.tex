\section{Problem Statement}

This report summarizes the current benchmark results for the nonlinear pendulum
swing-up problem across five solver items:
\begin{enumerate}
    \item Direct transcription in JAX (item 1)
    \item iLQR in JAX (item 2)
    \item MPPI/CEM in JAX (item 3)
    \item Direct transcription in \texttt{acados} (item 4)
    \item DDP/iLQR in \texttt{Crocoddyl} (item 5)
\end{enumerate}

The benchmark problem is the shared nonlinear pendulum from page 6 of the
project document. The state is $x = (\theta, \dot{\theta})$, the control is a
bounded torque input $u$, the initial state is the hanging equilibrium
$x_0 = (0, 0)$, and the target is the upright rest state
$x_{\mathrm{goal}} = (\pi, 0)$. We compare hard-constraint and soft-constraint
modes and evaluate scaling with horizon length $N \in \{20, 200\}$.

\section{Hardware Setup}

\subsection{GPU Items: Google Colab}

Items 1--3 were tested in a Google Colab GPU runtime using JAX with a CUDA
backend. In the Colab logs, these runs reported
backend: gpu and devices: (`cuda:0',). The exact NVIDIA GPU
model in Colab depends on the runtime assigned to the session, so it may vary
between runs unless it is recorded explicitly from \texttt{nvidia-smi}.

\subsection{CPU Items: Local Laptop}

Items 4--5 were tested on the local laptop:
\begin{itemize}
    \item machine: Apple MacBook Air
    \item chip: Apple M4
    \item architecture: \texttt{arm64}
    \item operating system: macOS 26.0.1
    \item available CPU threads reported by Python: 10
\end{itemize}

The CPU baselines use the local \texttt{acados} and \texttt{Crocoddyl}
installations, with \texttt{acados} built with OpenMP enabled.
